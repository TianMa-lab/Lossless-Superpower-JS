# systematic-debugging 技能

| name | description |
|------|-------------|
| systematic-debugging | 发现 Bug 时系统化调试 |

## 触发条件
- 测试失败
- 用户报告问题
- 系统异常

## 四阶段调试流程

### Phase 1: 复现
```
1. 获取完整的错误信息/堆栈
2. 确认复现步骤
3. 最小化复现用例
```

### Phase 2: 定位
```
1. 二分法缩小范围
2. 检查最近的改动
3. 隔离问题模块
```

### Phase 3: 根因分析
```
追问"为什么"至少 3 次：
- 为什么出错？→ 直接原因
- 为什么会有这个原因？→ 深层原因
- 为什么深层原因存在？→ 根本原因
```

### Phase 4: 修复与验证
```
1. 编写回归测试（先 RED）
2. 实现修复（GREEN）
3. 验证原问题解决
4. 验证无副作用
```

## 调试日志模板
```markdown
## Bug: [简短描述]
- 症状: [表现]
- 复现: [步骤]
- 根因: [原因]
- 修复: [方案]
- 验证: [测试]
```
